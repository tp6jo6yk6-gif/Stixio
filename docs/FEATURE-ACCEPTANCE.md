## 本次發現並修正

### 網格邊距／間距滑桿錯誤

原始程式會由控制項名稱推導不存在的顯示標籤 ID，例如把 `gridMarginX` 錯誤轉換成 `marginxValX`，因此操作滑桿時會發生 JavaScript 錯誤並中止設定。

已改為明確對照：

```js
const gridValueLabels = {
  gridMarginX: 'marginValX',
  gridMarginY: 'marginValY',
  gridGapX: 'gapValX',
  gridGapY: 'gapValY'
};
```

同時移除兩行無效且重複的 `state.layout === '1x1'` 判斷。

## 已知產品行為／後續改善

1. Sticker 官方命名目前仍把第 1、2 張貼圖指定為 `main.png` 與 `tab.png`。後續應改成由使用者指定代表圖，再另外衍生 main／tab，不要消耗正式貼圖。
2. 多張高解析圖片同時啟用羽化、侵蝕及外框時，主執行緒可能有明顯運算延遲；後續應加入工作佇列或 Web Worker。
3. 本次已驗證功能與引擎輸出路徑；實際上架前仍應使用一組真實貼圖素材做人工視覺檢查，包括白邊、細髮、透明漸層及安全留白。
