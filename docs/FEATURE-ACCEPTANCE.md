## 本次 Refine 發現並修正

### 手動 Delete 被 Chroma 開關一併停用

舊處理流程在 `chromaEnabled=false` 時會直接略過整段遮罩處理，導致「強制去背」筆刷看似有畫，最終輸出卻沒有透明效果。現在手動 Delete 與自動去背解耦，即使關閉自動去背仍會執行。

### Keep 遮罩會被後處理再次破壞

原本 Keep 僅在色鍵去背階段生效，之後的去雜點、侵蝕與羽化仍可能把使用者指定保留的像素刪除或變透明。現在所有自動處理完成後會再次套用手動遮罩：Keep 還原原始像素，Delete 強制透明，作為最終權威結果。

### Refine 缺少可驗收的工作視窗

正式 UI 現已加入遮罩原圖與成品雙預覽、筆刷游標、遮罩透明度、魔術動作、縮放平移、空白鍵操作、單張重算與套用狀態，並由 Playwright 直接操作 Canvas 驗證。

## 本次 Review 發現並修正

### 像素檢查引擎未接入正式工作流程

舊版雖有透明像素分析函式，但 Review 畫面只顯示縮圖與檔案大小。現在像素分析會產生可導覽的品質問題，並與每張 Frame、核准狀態及 Package 門檻連動。

### 預覽核准沒有失效機制

過去沒有正式核准狀態。現在 Frame 的裁切、Offset、遮罩或輸出設定一旦改動，舊核准會自動撤銷，避免編輯後仍沿用過期檢查結果。

### 匯出前缺少品質門檻

現在只有「所有選定輸出均已核准」且「沒有 Error」時才能進入 ZIP Package；Warning 會保留提示，但不強制阻擋核准。

### 預覽互動不應依賴樣式框架

Chromium 驗收發現定位、事件穿透與裁切若只寫在工具類別中，無樣式框架時可能失效。關鍵預覽機制現以必要 inline style 保證功能成立，Tailwind 僅負責視覺外觀。

## 已知產品行為／後續改善

1. 多張高解析圖片同時啟用羽化、侵蝕、外框與像素分析時，主執行緒仍可能有明顯運算延遲；後續可加入工作佇列或 Web Worker。
2. 自動化驗收已確認像素 Alpha 與互動狀態；實際發版前仍應使用真實貼圖素材人工檢查細髮、半透明漸層、白邊與複雜內孔。
3. Refine 遮罩與 Review 核准目前保存在工作階段記憶體；後續 Document／Project 儲存功能需將狀態序列化到 `.stixio` 與 Workspace Storage。
4. Review 已完成平台中立品質門檻；各目的地的數量限制、命名規則與檔案大小硬限制由後續 Destination Rules／Package 階段接管。
